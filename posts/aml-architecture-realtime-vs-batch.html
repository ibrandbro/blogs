AML Architecture: Real-Time vs Batch

One of the most common implementation questions in AML programs is not whether monitoring is needed, but how the architecture should be split between real-time APIs, event streams, and batch platforms.

Where real-time belongs

  • Onboarding decisions
  • Identity verification checks
  • Sanctions screening at critical decision points
  • Immediate transaction holds or blocks

Where batch still matters

  • Historical feature generation
  • Large-scale model scoring
  • Retrospective pattern detection
  • Reconciliations and reporting support

Strong AML architecture is rarely one or the other. The better design question is how to place the right control in the right execution model.
